# 🌐 Level 1.2: Networking Fundamentals - The Digital Pipes

Dalam *cyber security*, jaringan adalah medan perangnya. Kamu harus paham bagaimana data berpindah dari titik A ke titik B sebelum bisa mencegat atau mengamankannya.

---

## 1. Model OSI (7 Layers)
Bayangkan Model OSI sebagai standar bahasa yang digunakan komputer untuk berkomunikasi. Sebagai pemula, fokuslah pada 4 layer terbawah karena di situlah banyak "sihir" terjadi.



1. **Layer 1 (Physical):** Kabel, WiFi, Hub.
2. **Layer 2 (Data Link):** MAC Address (Identitas fisik perangkat).
3. **Layer 3 (Network):** IP Address (Alamat pengiriman data).
4. **Layer 4 (Transport):** TCP/UDP (Cara data dikirim, apakah mau cepat atau mau pasti sampai).

---

## 2. Alamat di Dunia Digital
Ada dua alamat utama yang wajib kamu pahami:
- **IP Address:** Ibarat alamat rumahmu (Contoh: `192.168.1.1`).
- **MAC Address:** Ibarat nomor rangka kendaraan, unik dan tidak berubah (Contoh: `00:1A:2B:3C:4D:5E`).

---

## 3. Port & Protokol (Pintu Masuk)
Komputer memiliki ribuan "pintu" virtual yang disebut **Port**. Setiap port biasanya menjalankan layanan tertentu.



| Port | Protokol | Kegunaan |
| :--- | :--- | :--- |
| 22 | **SSH** | Remote akses terminal secara aman. |
| 80 | **HTTP** | Web server (tidak aman). |
| 443 | **HTTPS** | Web server (aman/terenkripsi). |
| 21 | **FTP** | Transfer file. |
| 53 | **DNS** | Mengubah nama domain (google.com) jadi IP. |

---

## 4. Senjata Terminal untuk Jaringan
Gunakan perintah ini di Linux untuk melihat apa yang terjadi di jaringanmu:

- `ip a`: Melihat IP Address yang sedang kamu gunakan.
- `ping google.com`: Mengecek apakah kamu terhubung ke server tujuan.
- `traceroute google.com`: Melihat lompatan (hops) yang dilewati datamu untuk sampai ke tujuan.
- `ss -tuln` (atau `netstat`): Melihat port mana saja yang sedang "Terbuka" (Listening) di komputermu.
- `nmcli`: Mengelola koneksi jaringan via terminal.

---

## 5. Konsep Firewall
Firewall adalah satpam yang menjaga port. Di Linux, kita sering menggunakan **UFW (Uncomplicated Firewall)** atau **iptables**.
- `sudo ufw status`: Cek status firewall.
- `sudo ufw allow 22`: Membuka pintu SSH.
- `sudo ufw deny 80`: Menutup pintu HTTP.
