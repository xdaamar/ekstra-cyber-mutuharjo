# 🛡️ Level 1.1: User Management & Permissions - The Gatekeeper

Setelah menguasai navigasi, sekarang kamu harus belajar cara "mengunci pintu" sistem. Di Linux, segala sesuatu adalah file, dan setiap file memiliki pemilik serta izin akses.

---

## 1. Konsep User di Linux
Ada dua jenis user utama yang wajib kamu tahu:
1.  **Standard User:** User biasa dengan akses terbatas (untuk keamanan).
2.  **Root User (Superuser):** Tuhan-nya sistem. Bisa menghapus apa saja, melihat apa saja. Di terminal, user ini biasanya ditandai dengan simbol `#`, sedangkan user biasa dengan `$`.

### Perintah Cek Identitas:
- `whoami`: Mengetahui nama user yang sedang aktif.
- `id`: Melihat detail UID (User ID) dan GID (Group ID).
- `sudo`: (SuperUser DO) Menjalankan perintah dengan hak akses root.

---

## 2. Memahami File Permissions (rwx)
Saat kamu mengetik `ls -l`, kamu akan melihat deretan karakter seperti `-rwxr-xr--`. Mari kita bedah:



Struktur izin dibagi menjadi 3 kelompok:
1.  **Owner (User):** Orang yang membuat file.
2.  **Group:** Sekelompok user yang memiliki akses yang sama.
3.  **Others:** Semua orang lain di sistem.

Setiap kelompok punya 3 jenis izin:
- `r` (Read): Boleh membaca file/melihat isi folder. (Nilai: 4)
- `w` (Write): Boleh mengedit file/menambah file di folder. (Nilai: 2)
- `x` (Execute): Boleh menjalankan file sebagai program/masuk ke folder. (Nilai: 1)

---

## 3. Mengubah Izin dengan `chmod`
Kamu bisa menggunakan angka (Octal) untuk mengubah izin dengan cepat.
- **7** (4+2+1) = Full access (`rwx`)
- **6** (4+2) = Read & Write (`rw-`)
- **5** (4+1) = Read & Execute (`r-x`)
- **4** = Read only (`r--`)

### Contoh Perintah:
- `chmod 777 file.txt`: Semua orang bisa melakukan apa saja (SANGAT BERBAHAYA!).
- `chmod 600 rahasia.txt`: Hanya Owner yang bisa baca & tulis, orang lain dilarang keras.
- `chmod +x script.sh`: Memberikan izin eksekusi pada sebuah file script.

---

## 4. Mengubah Pemilik dengan `chown`
Kadang kamu perlu memindahkan kepemilikan file ke user lain.
- `sudo chown user_baru file.txt`: Mengubah pemilik file.
- `sudo chown user:group file.txt`: Mengubah pemilik sekaligus grupnya.

---

## 5. Kenapa Ini Penting untuk Cyber Security?
Salah satu teknik serangan yang populer adalah **Privilege Escalation**. Penyerang akan mencari file yang memiliki izin `w` (Write) bagi `Others` atau file milik `root` yang punya bit `SUID` aktif, lalu memanfaatkannya untuk menjadi admin.

## 🏁 Mini Challenge: The Secret Agent
Praktikkan ini di terminalmu:
1. Buat file bernama `pesan_rahasia.txt` berisi "Password bank saya adalah 12345".
2. Ubah izin file tersebut agar **hanya kamu** (Owner) yang bisa baca dan tulis, sedangkan Group dan Others tidak bisa melihat apa-apa (Izin: `rw-------`).
3. Cek hasilnya dengan `ls -l pesan_rahasia.txt`. Pastikan tampilannya `-rw-------`.
4. Coba buat file script sederhana `hello.sh` berisi `echo "Hello World"`. Coba jalankan dengan `./hello.sh`. Jika "Permission Denied", gunakan `chmod` untuk memberinya izin `execute`.
