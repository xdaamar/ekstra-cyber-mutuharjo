# 🕵️ Level 1.3: Services & Log Analysis - Tracking the Footprints

Selamat! Kamu sudah sampai di ujung fase fondasi. Sekarang kita akan belajar cara mengelola layanan sistem dan bagaimana cara melihat siapa saja yang mencoba masuk ke sistemmu.

---

## 1. Apa itu Service (Daemon)?
*Service* adalah program yang berjalan di latar belakang tanpa perlu interaksi user secara langsung. Contohnya: Web Server (Apache), Database (MySQL), atau SSH Server.

Di Linux modern, kita menggunakan **systemd** untuk mengelola layanan-layanan ini. Perintah utamanya adalah `systemctl`.

### Perintah Penting `systemctl`:
* `sudo systemctl start [service]`: Menjalankan layanan.
* `sudo systemctl stop [service]`: Menghentikan layanan.
* `sudo systemctl restart [service]`: Memulai ulang (biasanya setelah ganti konfigurasi).
* `sudo systemctl status [service]`: Cek apakah layanan sedang jalan atau mati.
* `sudo systemctl enable [service]`: Membuat layanan otomatis jalan saat komputer nyala (boot).

## 2. Mengintip Jejak Digital (Log Analysis)
Segala sesuatu yang terjadi di Linux dicatat dalam file **Log**. Jika seorang hacker masuk ke sistemmu, jejaknya pasti ada di sini (kecuali mereka sangat ahli dan menghapusnya).

Hampir semua log disimpan di direktori `/var/log/`.

### File Log yang Wajib Kamu Tahu:
1.  **`/var/log/auth.log`**: (Paling Penting!) Mencatat semua aktivitas login, termasuk percobaan login yang gagal (serangan Brute Force).
2.  **`/var/log/syslog`**: Catatan umum aktivitas sistem secara keseluruhan.
3.  **`/var/log/apache2/access.log`**: Jika kamu menjalankan web server, di sini tercatat siapa saja yang mengunjungi web-mu.

---

## 3. Cara Membaca Log Secara Real-Time
Kamu tidak bisa membaca log yang sangat panjang dengan `cat`. Gunakan cara "Pro" ini:

* **`tail -f /var/log/auth.log`**: Perintah ini akan memantau log secara *real-time*. Setiap kali ada yang mencoba login, terminalmu akan langsung memunculkan baris baru.
* **`journalctl -u ssh`**: Melihat log khusus untuk layanan tertentu (misal: SSH).
* **`grep "Failed password" /var/log/auth.log`**: Mencari semua percobaan login yang gagal.



---

## 4. Kenapa Ini Penting untuk Cyber Security?
Sebagai seorang *Security Professional*, kamu harus bisa membedakan mana aktivitas normal dan mana serangan. 

- **Aktivitas Normal:** User `admin` login pukul 09:00 pagi.
- **Aktivitas Mencurigakan:** Ada 1.000 percobaan login gagal dalam 1 menit dari alamat IP asing menggunakan user `root`. (Ini adalah indikasi serangan **Brute Force**).

## 🏁 Mini Challenge: The Log Investigator
Ayo praktikkan insting detektifmu:
1. Cek status layanan SSH di komputermu dengan `sudo systemctl status ssh` (atau `sshd`).
2. Buka satu jendela terminal lagi, lalu jalankan `sudo tail -f /var/log/auth.log` (jika di Ubuntu/Debian) atau `sudo journalctl -f`.
3. Di jendela terminal pertama, coba lakukan perintah `sudo su` dan masukkan password yang **salah** 2-3 kali.
4. Lihat apa yang muncul di jendela terminal kedua. Apakah kamu melihat catatan "authentication failure"?
5. Coba cari berapa banyak percobaan login yang gagal hari ini dengan perintah:
   `grep "Failed password" /var/log/auth.log | wc -l`
